## 2. Компоненты системы

### 2.1 Клиентские приложения
- **Веб-приложение**: Основной интерфейс для управления домом (React + Vite)
- **Мобильное приложение**: Для уведомлений и быстрого доступа (React Native)
- **USSD-интерфейс**: Для экстренного управления без интернета

### 2.2 Backend сервисы
- **API Gateway**: Маршрутизация запросов, аутентификация (FastAPI)
- **WebSocket Server**: Real-time обновления состояния устройств
- **Scheduler**: Фоновые задачи, автоматизация, проверки (APScheduler)
- **Notification Service**: Интеграция с SMS/Push API оператора

### 2.3 Бизнес-логика
- **Device Manager**: Управление всеми типами устройств
- **Gamification Engine**: Система уровней, очков, достижений
- **Security Monitor**: Мониторинг безопасности, тревоги
- **Analytics Engine**: Сбор и анализ данных, отчёты

### 2.4 Хранение данных
- **SQLite**: Основная база данных
- **Файловая система**: Снимки камер, логи
- **Облако оператора**: Резервное копирование

## 3. Поток данных

```
Пользователь → Веб-приложение → API Gateway → Бизнес-логика → База данных
                                      ↓
                              WebSocket → Real-time обновления
                                      ↓
                              Scheduler → Автоматизация
                                      ↓
                              Telecom API → SMS/Push уведомления
```

## 5. API Endpoints

### Управление устройствами
- `GET /api/devices/` — список устройств
- `POST /api/devices/{id}/toggle` — вкл/выкл
- `GET /api/lights/` — светильники
- `POST /api/lights/{id}/brightness` — яркость
- `GET /api/climate/` — климат
- `POST /api/climate/{id}/temperature` — температура

### Подростковый дашборд
- `GET /api/teen/dashboard/{member_id}` — дашборд
- `GET /api/teen/tasks` — задачи
- `POST /api/teen/tasks/{id}/complete/{member_id}` — выполнить
- `POST /api/teen/focus/start` — начать фокус
- `GET /api/teen/home-status` — статус дома

### Достижения
- `GET /api/achievements/` — список достижений
- `GET /api/achievements/member/{member_id}` — достижения пользователя
- `POST /api/achievements/check/{member_id}` — проверить прогресс

### Семейные соглашения
- `GET /api/agreements/` — список соглашений
- `POST /api/agreements/` — создать соглашение
- `POST /api/agreements/{id}/agree/{member_id}` — согласиться

### Телеком-интеграция
- `POST /api/telecom/notifications` — создать уведомление
- `POST /api/telecom/notifications/{id}/send` — отправить
- `POST /api/telecom/energy-saving` — записать экономию
- `GET /api/telecom/energy-saving/summary` — сводка

### WebSocket
- `WS /api/ws/events` — real-time обновления

## 6. Интеграция с оператором

### 6.1 SMS Gateway
- Уведомления о забытых приборах
- Экстренные оповещения о безопасности
- Напоминания о задачах и ритуалах

### 6.2 Push API
- Мгновенные уведомления в приложении
- Обновления статуса устройств
- Результаты выполнения задач

### 6.3 Internet Control
- Режим «Фокус» с ограничением развлекательных сайтов
- Ночное замедление интернета
- Родительский контроль через API

### 6.4 USSD-коды
- `*123*1#` — статус дома
- `*123*2#` — экстренное выключение
- `*123*3#` — получение уведомлений

## 7. Безопасность

### 7.1 Аутентификация
- JWT токены для API
- Ролевой доступ (подросток/родитель)
- Двухфакторная аутентификация

### 7.2 Защита данных
- Шифрование чувствительных данных
- Резервное копирование в облако оператора
- Логирование всех действий

### 7.3 Приватность
- Прозрачные правила сбора данных
- Возможность отключения отслеживания
- Локальное хранение основной информации
